What is Machine Learning?

Machine Learning describes the ‘principles, methods and algorithms for learning and prediction on the basis of past evidence’. Some examples of Machine Learning applications in our daily lives are Apples’ Siri (speech recognition), email spam filters (text filtering) or autonomous vehicles.

A basic Machine Learning framework.

Machine Learning Methodologies

Supervised Learning is concerned with the task of inferring a function from a set of labled training data. These datasets include both the input as well as the desired output. Thus, the algorithm learns how input affects output and enables prediction.

Unsupervised Learning describes methods that aim at finding hidden structure in unlabled datasets. Since the model is not provided with the correct results during training, there is no error or reward process to evaluate the potential solution.

Machine Learning models estimate a function and its parameters. The concrete method depends on the nature of what is to be predicted and includes approaches such as: * Regression (Prediction of real, continuous numbers) * Classification (Prediction of discrete clas lables) * Clustering (Finding structure in data by clustering points according to their proximities)

K-means clustering

Our goal is to assign our data points to a predefined number of clusters, where each point belongs to the cluster with the closest mean. Initially, we generate random points to serve as cluster means, assign our observations, compute new cluster means and repeat this process until convergance. We are now going through this process in R:

We now plot the obtained results to gain first visual insights.

plot(countries, col = km$cluster)

Depending on the number of clusters, the k-means algorithm can be rather computationally expensive. We must cope with a trade-off between explanatory power and computational efficiency. A good tool to investigate this relation and help with decision making is the elbow plot, plotting the number of clusters against the explained variance in our model.

ev <- c()
for (i in 1:15) {
km <- kmeans(countries, i, nstart = 10)
ev[i] <- sum(km$betweenss)/km$totss
}
plot(1:15, ev, col="red", lwd=2, type = "l", xlab = "Number of Clusters", ylab = "Explained Variance")

We usually select a number of clusters k, so that adding another cluster does not result in explicitly better modelling of our data.

Artificial Neural Networks

Artificial Neural Networks (ANN) are computational models inspired by the central nervous system. It computes a function by feeding information through a network of neurons. Neurons are arranged in a set of layers, reaching from the input layer (recipient of the input data) over varied numbers of hidden layers (connecting input and output) to an output layer (obtaining the response values). The input of each neuron in the network is the weighted sum of all previous neurons.

In R, we start by loading the required libraries and access the dataset for this exercise, this time provided by the R package caret. We split the dataset into a training (80%) and a testing (20%) subsample.

#Loading required libraries nnet and ggplot2 (for dataviz)
library(ggplot2)
library(nnet)
#Accessing credit scores
library(caret)
data(GermanCredit)
#Split data into index subset for training (80%) and testing (20%) instances
inTrain <- runif(nrow(GermanCredit)) < 0.2

The dataset GermanCredit includes credit approval data as well as attributes characterising credit applications, such as income, gender, housing situation or credit purpose. We seek a function to find patterns in the dataset that helps with predicting credit application outcomes, as given by the column Class.

We train the ANN using the R command nnet, specifiying the number of hidden layer units, maximum iterations, initial random weights and weight decay.

#Train neural network with n nodes in the hidden layer via
#nnet(formula, data=d, size=n ...)
#Output variable: Class (binary 'accept credit application y/n')
nn <- nnet(Class ~ ., data=GermanCredit[inTrain,],
size=15, maxit=100, rang=0.1, decay=5e-4)

We use the results obtained from the ANN to predict credit application outcomes and compare the prediction with the actual values. We extract the confusion matrix, providing us with true positive (correctly predicted good outcome), true negative (correctly predicted bad outcome), false positive (falsely predicted good outcome) and false negative (falsely predicted bad outcome) values of our prediction. Using these numbers, we can compute model accuracy (\(ACC=(TP+TN)/(P+N)\)) and precision (\(PRE=TP/(TP+FP)\)).

#Predict credit scores via 
#predict(nn, test, type="class")
pred <- predict(nn, GermanCredit[-inTrain,],
type="class")
#Extract confusion matrix via 
#table(pred=pred_classes, true=true_classes)
cm_nn <- table(pred=pred, true=GermanCredit$Class[-inTrain])
cm_nn
##       true
## pred   Bad Good
##   Bad   33   24
##   Good 267  675
#Calculating accuracy
(cm_nn[1,1]+cm_nn[2,2])/
(cm_nn[1,1]+cm_nn[1,2]+cm_nn[2,1]+cm_nn[2,2])
## [1] 0.7087087
#Alternative that works also for multi-class data
sum(diag(cm_nn))/sum(sum(cm_nn))
## [1] 0.7087087
#Calculating precision
cm_nn[1, 1]/(cm_nn[1, 1] + cm_nn[1, 2])
## [1] 0.5789474
#Plot confusion matrix using ggplot 2
cm_nn_plot <- as.data.frame(cm_nn)
ggplot(data =  cm_nn_plot, mapping = aes(x = true, y = pred)) +
  geom_tile(aes(fill = Freq), colour = "white") +
  geom_text(aes(label = sprintf("%1.0f", Freq)), vjust = 1) +
  scale_fill_gradient(low = "light blue", high = "dark blue") +
  theme_bw() + theme(legend.position = "none")

This model shows an accuracy of around \(0.7\), impyling that approximately 70% of the credit application outcome predictions are accurate.

Support Vector Machines

A Support Vector Machine (SVM) is a linear seperator technique, that aims at maximizing the margins between the linear seperator and the closest respective data points, called support vectors. We are using the same dataset and training / testing subsamples as in the previous exercise on ANN.

#Loading required library e1071
library(e1071)
#Split data into index subsets for training (80%) and testing (20%) instances
inTrain <- runif(nrow(GermanCredit)) < 0.2

We train a SVM in R, calling the svm command and specifiying machine type and variable scaling.

#Train support vector machine for classification via svm(formula, data=d, type="C-classification")
#First row gives predicted outcomes, second are actual (true) values
model <- svm(Class ~ ., data=GermanCredit[inTrain,], type="C-classification", scale=FALSE)
#Predict credit scores for testing instances test via predict(svm, test)
pred <- predict(model, GermanCredit[-inTrain, ])
head(cbind(pred, GermanCredit$Class[-inTrain]))

The accuracy for the SVM approach is approximately \(0.75\).

Receiver Operating Characteristic (ROC)

The ROC illustrates the performance of binary classifiers, as their discrimination threshold is varied. ROCs can be used to compare the quality of different models by comparing their respective areas under the curve (AUCs).

#Load required library pROC
library(pROC)
#Perform prediction and retrieve decision values dv
pred <- predict(model, GermanCredit[-inTrain,], decision.values=TRUE)
dv <- attributes(pred)$decision.values
#Plot ROC curve
plot.roc(as.numeric(GermanCredit$Class[-inTrain]), dv, xlab = "1 - Specificity")

The grey diagonal line represents random guessing (50% accuracy) and every ROC above the line implies an model-induced improvement over a random guess.
