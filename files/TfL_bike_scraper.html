Mining London cycle hire data in R using PhantomJS

Konstantin Klemmer

October 15, 2017

Building web scrapers in R can sometimes be tricky, especially when the goal-webpage runs with JavaScript or has a delayed load. This way, ordinary HTML scrapers may be unsufficient. However, there is a very flexible workaround proposed by Florian Teschner (https://flovv.github.io/). Today, I want to apply this approach to mine the extensive and highly granular TfL cycle hire dataset.

The TfL cycle hire data is available via a massive csv dump from the TfL open data portal (http://cycling.data.tfl.gov.uk/). So this tutorial might come handy to you whenever you want to avoid downloading large file libraries by hand. Let’s get going:

Setting up the environment

The programme is rather straightforward and solely requires the rvest package (with dplyr and data.table for data handling), which enables an R based web scraper. We also set the working directory of our choice.

require(rvest)
require(dplyr)
require(data.table)
setwd("~/Data")

We now follow Florian Teschners steps to initialise the scraper (His tutorial can be found here: https://flovv.github.io/Scrape-JS-Sites/). Note that this requires the installation of PhantomJS, a browser optimised for automated web interactions (PhantomJS can be downloaded here: http://phantomjs.org/). It is important to place the phantomjs.exe file in your working directory, i.e. ~\Data. Once this is done we download the prepared JavaScript scraper (https://gist.githubusercontent.com/flovv/91453712e8a6ba957e63/raw/70e408ee71b4b28f09461245e9b65e45141cb06a/scrape_final.js) and also save it in the workind directory.

Scraping London cycle hire data

We now prepare a R function to run the scraper.

#CREATE scrapeJSSite FUNCTION
scrapeJSSite <- function(x,y){ #x=goal URL, y=delay in milliseconds
  #ADAPT ORIGINAL PHANTOMJS SCRAPER GOAL URL
  url <- paste0(x)
  lines <- readLines("scrape_final.js") #Scraper JS file must be in working directory!
  lines[1] <- paste0("var url ='", url ,"';") #Define goal URL (x)
  #lines[14] <- paste0("    }, ", y ,");") #Define delay (y)
  writeLines(lines, "scrape_final.js")
  #DOWNLOAD WEBSITE VIA JS SCRAPER
  system("phantomjs scrape_final.js")
  #USE RVEST TO SCRAPE DELAYED WEBSITE
  pg <- read_html("1.html")
  return(pg)
}

We can now run the scraper on our desired webpage. We define the x variable of the custom function as the goal webpage and the y variable as the loading delay.

wp <- "http://cycling.data.tfl.gov.uk" #Goal URL
wp_html <- scrapeJSSite(wp,500) #Run Scraper with 5000ms delay

wp_html now contains the scraped website HTML as extracted after a 5000 millisecond delay. We can now collect all file URLs from the webpage, looking for the according HTML tags. We save all links in the filelinks dataframe.

filelinks <- as.data.frame( #SELECT LINKS ONLY
  wp_html %>%
    html_nodes("a") %>%
    html_attr("href")
)
colnames(filelinks) <- "filename"

Download and read cycling data

We now use the file link database to download exactly the data we are searching, for instance all cycle trips from 2016. Luckily, TfL names each file according to the period it covers–ending with the observation year. We can thus easily select all files ending on 16.csv to download. Obviously, this process might take some time, depending on how many files we select.

fl16 <- filelinks %>% filter(str_detect(filename,"16.csv")) #Select links from 2016
fl16 <- fl16[1:length(fl16$filename),]
dir.create(file.path("~/Data/bike/", "2016")) #Create subdirectory for download
setwd("~/Data/bike/2016") #Set working directory
for (i in fl16) { #Use for-loop to download all selected files
  download.file(i, destfile=basename(i),mode="wb") #wb mode must be enabled!
}

We are almost done! In the last step, we read the downloaded files into R using the fread function from the data.table package.

setwd("~/Data") #Reset working directory
fns <- list.files("bike/2016") #List names of downloaded files
setwd("~/Data/bike/2016") #Adjust the temporary working directory
bt2016 <- rbind.fill(lapply(fns, fread, header=TRUE)) #Read and bind trips in one data frame
setwd("~/Data") #Reset working directory

We have now imported all TfL cycle hire trips from 2016, just using a few lines of code. Happy crafting!
